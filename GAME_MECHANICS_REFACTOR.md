# 游戏机制重构总结

## 🎯 新的游戏机制

### 1. **每天分配去向系统**
- 船员只能在三个地点中选择一个：
  - **宿舍 (QUARTERS)**: 休息恢复HP和能量
  - **训练模拟器 (SIMULATOR)**: 训练获得经验值
  - **任务指挥部 (MISSION_CONTROL)**: 待命准备参加战斗
  - **待命 (UNASSIGNED)**: 未分配状态

### 2. **战斗冷却机制**
- 参与战斗后，船员必须间隔 **1天** 才能再次参加战斗
- 新招募的船员可以立即使用（`daysSinceLastCombat = 2`）
- UI会显示冷却状态：
  - ⏰ "还需休息 X 天才能参加战斗"
  - ✅ "可以参加战斗"

### 3. **招募船员系统**
- 在待命界面添加了 **"👥 招募新船员"** 按钮
- 招募成本：**100资源**
- 随机生成：
  - 职业（从所有职业中随机选择）
  - 名字（中文姓氏+名字组合）
- 新船员默认分配到"待命"状态
- 初始资源：**200**

### 4. **战斗文本增大**
- 战斗日志字体从 **14sp** 增大到 **18sp**
- 提高可读性

## 🎮 游戏流程说明

### 每日流程：

1. **调度阶段 (SCHEDULING)**
   - 玩家为每个船员分配去向：
     - 点击船员选中
     - 点击"宿舍"/"训练"/"任务部"/"待命"按钮分配
   - 可以招募新船员（消耗100资源）
   - 查看船员的战斗冷却状态

2. **进展阶段 (PROGRESSION)**
   - 点击"下一阶段"进入
   - 自动处理：
     - 宿舍船员：恢复HP +20, 能量 +25
     - 训练船员：获得XP +20, 能量 -10
     - 所有船员：战斗冷却天数递减
     - 天数 +1

3. **任务选择阶段 (MISSION_SELECTION)**
   - 选择要执行的任务
   - 组建小队（最多5人）
   - **只能选择冷却完成的船员**
   - 点击"开始任务"进入战斗

4. **战斗阶段 (COMBAT)**
   - 回合制战斗
   - 战斗结束后，所有参战船员标记为"已参战"
   - 需要休息1天才能再次参战

## 🔍 关键设计决策

### 1. 为什么移除行动次数限制？
- **原问题**：每人3次行动让宿舍、训练模拟器、任务指挥部失去意义
- **解决方案**：改为每天分配一次去向，更符合策略游戏的设计

### 2. 为什么添加战斗冷却？
- **游戏平衡**：防止同一批强力船员连续作战
- **策略深度**：玩家需要合理轮换船员
- **真实感**：模拟战斗后的疲劳和恢复

### 3. 为什么添加招募系统？
- **可扩展性**：玩家可以拥有更多船员
- **资源管理**：需要在招募和资源积累之间权衡
- **多样性**：不同职业的船员提供不同的战术选择

### 4. 为什么增大战斗文本？
- **用户体验**：原14sp字体在小屏幕上难以阅读
- **无障碍**：提高可读性，减少视觉疲劳

## ⚠️ 注意事项

1. **存档兼容性**：旧的存档可能没有`resources`字段，加载时会默认为0
2. **Gson序列化**：`resources`字段会自动被Gson序列化和反序列化
3. **战斗冷却计算**：`advanceDay()`只在进展阶段调用，确保冷却正确递减
